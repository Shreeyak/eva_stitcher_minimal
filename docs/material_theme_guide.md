# Material 3 Theming in Flutter — How It Works in This App

## Overview

This app uses the Material Theme Builder–generated `MaterialTheme` class
(`lib/theme/material_theme_<name>.dart`) together with a `createTextTheme()` helper
(`lib/theme/theme_util.dart`).  All widgets read colors from
`Theme.of(context).colorScheme` — no hard-coded color constants anywhere.

---

## Tuning animations

M3 ships `InkSparkle` as the default splash on Android — it has a long sparkle
dissipation tail (~600–900 ms) that makes chip/button taps feel sluggish.
Control it in the `darkTheme` (and `theme`) `.copyWith(...)` in `main.dart`:

```dart
darkTheme: materialTheme.dark().copyWith(
  snackBarTheme: const SnackBarThemeData(shape: snackBarShape),

  // Cut FilterChip background-fill transition (default ~200 ms → 80 ms).
  chipTheme: const ChipThemeData(
    animationDuration: Duration(milliseconds: 80),
  ),

  // Replace InkSparkle (long tail) with a short circular ripple.
  // Use NoSplash.splashFactory for no ripple at all.
  splashFactory: InkRipple.splashFactory,
),
```

`splashFactory` is app-wide (affects all `InkWell`, buttons, chips).
`chipTheme.animationDuration` is chips only.

```dart
// in main.dart, inside your ThemeData.copyWith(...)
chipTheme: ChipThemeData(
  animationDuration: const Duration(milliseconds: 80),
  surfaceTintColor: Colors.transparent, // optional: suppress tint layer too
),
// and then on the FilterChip itself in _CamSettingsChip:
// splashFactory: NoSplash.splashFactory,  ← per-widget override
```

## What the generated files contain

### `lib/theme/material_theme_<name>.dart`

Generated by [Material Theme Builder](https://m3.material.io/theme-builder).
Contains:

- **`MaterialTheme`** class — one constructor, six `ThemeData` builders:
  `light()`, `lightMediumContrast()`, `lightHighContrast()`,
  `dark()`, `darkMediumContrast()`, `darkHighContrast()`.
- **`theme(ColorScheme)`** — private builder that applies the scheme to a
  `ThemeData` with `useMaterial3: true`.
- **`ExtendedColor` / `ColorFamily`** — helpers for any custom seed colours
  added in Material Theme Builder beyond the default set.

This is the **single file to replace** when switching to a different palette.
The class name `MaterialTheme` stays the same; all widgets continue to work.

### `lib/theme/theme_util.dart`

Contains `createTextTheme(bodyFont, displayFont)` which merges two
Google Fonts into one `TextTheme`:

- `displayFont` is used for headlines, display, and title styles.
- `bodyFont` is used for body, label, and caption styles.
- The result is passed to `MaterialTheme(textTheme)`.

## ColorScheme slot → UI role mapping

| Slot | Typical UI role | Where used in this app |
| --- | --- | --- |
| `surface` | Scaffold / canvas background | Scaffold, `CanvasView` |
| `surfaceContainer` | Toolbars, panels, drawers | `LeftToolbar`, `CameraSettingsDrawer`, `BottomInfoBar` |
| `surfaceContainerHighest` | Elevated cards / secondary panels | Grid major lines |
| `primary` | Active highlights, selected state | Active `SideButton` border, active `FilterChip`, scanning dot |
| `primaryContainer` | Subtle active background | Active `SideButton` bg, Auto-mode button bg |
| `onPrimary` | Content placed on primary | Text/icons inside active chips |
| `secondary` / `secondaryContainer` | M3 chip selected state | `FilterChip` selected fill (auto-applied) |
| `tertiary` | Automatic/AWB indicator | Auto-mode value label tint |
| `outline` | Muted text, disabled state, idle dot | Disabled icons, `IDLE` status dot |
| `outlineVariant` | Dividers, borders | `Divider`, chip borders, WB panel border |
| `onSurface` | Primary text | Ruler dial icons |
| `onSurfaceVariant` | Secondary text | `BottomInfoBar` values, `SideButton` default icon |

## Widget usage pattern

In any widget where `BuildContext` is available:

```dart
@override
Widget build(BuildContext context) {
  final cs = Theme.of(context).colorScheme;
  // Use cs.primary, cs.surfaceContainer, cs.outlineVariant, etc.
}
```

In `CustomPainter` subclasses — which have no `BuildContext` inside `paint()` —
pass the needed colors as constructor parameters from the parent widget:

```dart
// Parent widget:
CustomPaint(
  painter: _GridPainter(
    minorColor: cs.surfaceContainer,
    majorColor: cs.surfaceContainerHighest,
    crosshairColor: cs.outlineVariant,
  ),
)

// Painter:
class _GridPainter extends CustomPainter {
  final Color minorColor;
  final Color majorColor;
  final Color crosshairColor;
  const _GridPainter({...});
  // use the colors directly in paint()
}
```

## Swapping themes

1. Export a new theme from [Material Theme Builder](https://m3.material.io/theme-builder).
2. Replace **only** `lib/theme/material_theme.dart` with the new `theme.dart` export.
3. Hot-restart — every surface in the app recolors automatically.
4. Optionally update the font pair in `main.dart`:
   `createTextTheme(context, 'Roboto', 'Noto Sans')`.

No widget files need to change.

---

## M3 auto-theming — when to omit `color:` arguments

With `useMaterial3: true` the following widgets derive their colors from
`colorScheme` automatically.  **Do not pass explicit `color:` arguments** — it
defeats the auto-theming and breaks theme-swap:

| Widget | Auto slot |
| --- | --- |
| `Divider` / `VerticalDivider` | `outlineVariant` |
| `FilterChip` unselected | `outline` border |
| `FilterChip` selected | `secondaryContainer` fill |
| `FilledButton` | `primary` fill, `onPrimary` text |
| `OutlinedButton` | `outline` border, `primary` text |
| `SegmentedButton` selected | `secondaryContainer` |
| `InkWell` ripple | `primary` with opacity |
| `SnackBar` | only the shape needs a `.copyWith` override |

## How this app initializes the theme (`main.dart`)

```dart
@override
Widget build(BuildContext context) {
  final textTheme = createTextTheme(context, 'Roboto', 'Noto Sans');
  final materialTheme = MaterialTheme(textTheme);
  return MaterialApp(
    theme: materialTheme.light(),
    darkTheme: materialTheme.dark().copyWith(
      snackBarTheme: const SnackBarThemeData(...),  // app-specific shape only
    ),
    themeMode: ThemeMode.dark,     // camera app is always dark
    ...
  );
}
```
